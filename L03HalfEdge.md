## 1. 引言

在计算共形几何中，**三角网格** (Triangle Mesh) 是离散化的核心表示形式。现实世界的几何数据——无论是扫描的3D模型、医学图像重建的器官表面，还是CAD软件设计的工业零件——最终都需要用三角网格来近似表示。

**为什么选择三角网格？**

1. **普遍性**: 任意光滑曲面可以用三角网格逼近
2. **简单性**: 每个面都是最简单的多边形（三角形）
3. **灵活性**: 可以局部细化或粗化
4. **计算友好**: 三角形是最稳定的几何元素

**数据结构的重要性**：

在共形几何算法中，我们需要频繁执行：
- 遍历邻接元素（顶点、边、面）
- 计算曲率、通量、梯度
- 执行重网格化、参数化操作

一个好的数据结构可以使这些操作 $O(1)$ 或 $O(\log n)$ 完成，而差的数据结构可能导致 $O(n)$ 甚至无法实现。

## 2. 三角网格表示

### 2.1 三角网格的定义

**定义**：三角网格 $M = (V, E, F)$ 由三部分组成：

- **顶点集** $V = \{v_1, v_2, \ldots, v_n\}$，$n = |V|$ 为顶点数
- **边集** $E \subset V \times V$，$m = |E|$ 为边数
- **面集** $F = \{f_1, f_2, \ldots, f_k\}$，$k = |F|$ 为面数

每条边连接两个顶点，每个面由三条边围成。

### 2.2 网格的组合拓扑

**流形网格** (Manifold Mesh) 的定义：

- **边流形**: 每条边恰好属于 1 个面（边界边）或 2 个面（内部边）
- **顶点流形**: 每个顶点处存在一个"扇"(star)，其面按环形排列无分叉
- **无悬挂边/面**: 每条边被恰好两个面共享（边界边除外）

**闭曲面**: 没有边界边的流形曲面。

**有边界曲面**: 有边界边的曲面，边界边只属于一个面。

### 2.3 常用的网格表示方法

| 方法 | 顶点 | 邻接关系 | 优点 | 缺点 |
|------|------|---------|------|------|
| 面-顶点表 (FV) | 坐标数组 | 面列表含顶点索引 | 简单直观 | 邻接查询慢 |
| Winged-Edge | 边为核心 | 边含两顶点+两面+四边 | 完整连通 | 实现复杂 |
| 半边 (Half-Edge) | — | 半边双向链表 | 面向对象，效率高 | 边界处理需特殊注意 |
| 入边 (Indexed) | 顶点+索引 | 紧凑存储 | 内存高效 | 操作不够直观 |

## 3. 半边数据结构定义

### 3.1 半边的基本概念

**半边** (Half-Edge) 是有向边。一条无向边 $\{u, v\}$ 被分解为两条有向半边：
- **正向半边**: $h = (u \to v)$
- **反向半边**: $\bar{h} = (v \to u)$

每个半边 $h$ 记录：
- **起点** $\text{vertex}(h) = u$
- **对边** $\text{pair}(h) = \bar{h}$
- **下一条半边** $\text{next}(h)$：同面中沿逆时针方向的下一条半边
- **前一条半边** $\text{prev}(h)$：同面中沿逆时针方向的上一条半边
- **面** $\text{face}(h)$：该半边所属的面

### 3.2 半边结构的数据表示

```cpp
// 半边结构
struct HalfEdge {
    Vertex*  vertex;    // 半边的起点
    HalfEdge* pair;    // 配对半边（反向）
    HalfEdge* next;    // 同面下一条半边
    HalfEdge* prev;    // 同面前一条半边
    Face*     face;    // 所属面
};

// 顶点结构
struct Vertex {
    Vector3   position; // 位置坐标
    HalfEdge* halfedge; // 从该顶点出发的任意一条半边
    // 可扩展属性：UV坐标、法向量、曲率等
};

// 面结构
struct Face {
    HalfEdge* halfedge; // 指向该面任意一条半边
    // 可扩展属性：法向量、面积等
};
```

### 3.3 半边循环不变量

对于流形网格，以下恒等式始终成立：

1. **配对双重性**: $\text{pair}(\text{pair}(h)) = h$
2. **循环闭合**: $\text{next}(\text{next}(\text{next}(h))) = h$（三角形恰好三条半边）
3. **面-半边对应**: $\text{face}(h) = \text{face}(\text{next}(h)) = \text{face}(\text{next}(\text{next}(h)))$
4. **顶点一致性**: $\text{vertex}(h) = \text{vertex}(\text{prev}(\text{pair}(h)))$

## 4. 半边结构的连通关系

### 4.1 顶点的邻接关系

给定顶点 $v$，所有以 $v$ 为起点的半边构成了 $v$ 的 **出边扇** (outgoing half-edge star)：

$$\text{star}(v) = \{h \mid \text{vertex}(h) = v\}$$

**顶点的1-环邻域**（相邻顶点）：

$$\mathcal{N}_1(v) = \{\text{vertex}(\text{pair}(h)) \mid h \in \text{star}(v)\}$$

**顶点的2-环邻域**（相邻顶点及其邻居）：

$$\mathcal{N}_2(v) = \mathcal{N}_1(v) \cup \{\text{vertex}(\text{pair}(\text{next}(h))) \mid h \in \text{star}(v)\}$$

### 4.2 面的邻接关系

给定面 $f$，该面的三条半边按逆时针排列：

$$h_0 = \text{halfedge}(f), \quad h_1 = \text{next}(h_0), \quad h_2 = \text{next}(h_1)$$

**相邻面**: $\text{neighbor}(f) = \{\text{face}(\text{pair}(h_i)) \mid i = 0, 1, 2\}$

### 4.3 边的属性

每条无向边 $\{u, v\}$ 对应一对半边 $h = (u \to v)$ 和 $\bar{h} = (v \to u)$。

**边的顶点**: $\text{endpoints}(h) = \{\text{vertex}(h), \text{vertex}(\text{pair}(h))\}$

**边所属的面**: 若 $\text{face}(h)$ 和 $\text{face}(\text{pair}(h))$ 都非空，则为内部边；否则为边界边。

## 5. 遍历操作

### 5.1 顶点遍历

**遍历顶点所有出边** (Walk Outgoing):

```
HalfEdge* h = v->halfedge;
do {
    // 处理半边 h
    h = pair(next(h));  // 沿逆时针方向跳到下一条出边
} while (h != v->halfedge);
```

**遍历顶点的相邻面** (顶点1-环):

```
HalfEdge* h = v->halfedge;
do {
    Face* f = face(h);  // 处理面 f
    h = pair(next(h));
} while (h != v->halfedge);
```

### 5.2 面遍历

**遍历面的所有顶点** (逆时针):

```
HalfEdge* h = f->halfedge;
Vertex* v0 = vertex(h);
Vertex* v1 = vertex(next(h));
Vertex* v2 = vertex(next(next(h)));
// v0, v1, v2 即为该面的三个顶点
```

**遍历所有面**: 简单地遍历面数组或半边表。

### 5.3 边遍历

**遍历所有边**（每条无向边恰好一次）:

由于每条无向边对应两条半边，我们只遍历所有半边中"配对半边索引大于自身索引"的那些：

```
for (int i = 0; i < numHalfEdges; i++) {
    if (i < pair(i)) {
        // 处理第 i 条半边对应的无向边
    }
}
```

### 5.4 边界遍历

对于有边界的网格，**边界边**（只有一面相邻）需要特殊处理：

**识别边界半边**: $\text{face}(h) \neq \emptyset$ 且 $\text{face}(\text{pair}(h)) = \emptyset$

**遍历边界环路**:

```
// 找到一条边界半边
HalfEdge* h = findBoundaryHalfEdge();
HalfEdge* start = h;
do {
    // 处理边界半边 h
    h = next(pair(h));  // 沿边界前进
} while (h != start);
```

## 6. 关键算法实现

### 6.1 网格构建

从面-顶点表构建半边结构：

```
Input: vertices V[], faces F[] (每个面含三个顶点索引)
Output: 半边网格结构

1. 为每条有向边 (face[i][j] -> face[i][(j+1)%3]) 创建一条半边
2. 建立哈希表: 半边键 -> 半边索引
3. 填充每条半边的: vertex, face, next
4. 配对半边: 查找 (v_next -> v) 配对 (v -> v_next)
5. 若某半边无配对，则为边界半边
```

### 6.2 离散曲率计算（平均曲率）

顶点的**平均曲率**可以通过1-邻域内的角度亏量计算：

$$\kappa(v) = \frac{1}{A(v)} \sum_{i} \left( \pi - \theta_i \right) \cdot \frac{l_i}{2}$$

其中：
- $\theta_i$ 是相邻三角形在顶点 $v$ 处的内角
- $l_i$ 是对边的长度
- $A(v)$ 是顶点 $v$ 的 **Voronoi 面积**（对偶面积）

### 6.3 梯度与拉普拉斯算子

**离散梯度** (Cotangent Weight Laplacian):

$$(\nabla_\mathcal{M} u)(v) = \frac{1}{2A(v)} \sum_{i \in \mathcal{N}_1(v)} \cot \alpha_{ij} (u(v_j) - u(v_i)) \vec{e}_{ij}$$

其中 $\alpha_{ij}$ 是三角形中对边所对的角，$\vec{e}_{ij}$ 是边 $(i,j)$ 方向的单位向量。

**离散拉普拉斯-贝尔特拉米算子**:

$$(\Delta_\mathcal{M} u)(v) = \frac{1}{2A(v)} \sum_{i \in \mathcal{N}_1(v)} \cot \alpha_{ij} (u(v_j) - u(v_i))$$

这是共形几何中最重要的离散算子之一，在调和映射、热核、Ricci 流等算法中无处不在。

### 6.4 法向量计算

**面法向量** (由顶点叉积确定方向):

$$\vec{n}_f = \frac{(v_1 - v_0) \times (v_2 - v_0)}{\|(v_1 - v_0) \times (v_2 - v_0)\|}$$

**顶点法向量** (常用两种):

1. **简单平均**: $\vec{n}_v = \sum_{f \in \text{star}(v)} \vec{n}_f$
2. **面积加权**: $\vec{n}_v = \sum_{f \in \text{star}(v)} A_f \vec{n}_f$

## 7. 与共形几何的关系

### 7.1 离散度量

三角网格的 **离散黎曼度量** 通过边长决定：

$$g_{ij} = |v_i - v_j|^2$$

满足三角不等式的边长集合 $\{l_e\}$ 决定了一个 **Euclidean triangulated surface**。

### 7.2 离散共形结构

共形几何的离散化需要将 **角度** 作为基本几何不变量：

- **内角** $\alpha_{ijk}$ 由边长通过余弦定理决定
- **共形因子** $e^{2u}$ 是顶点处的局部缩放因子
- **离散高斯曲率** $K(v)$ 由角度亏量定义

$$K(v) = \frac{2\pi - \sum_{i} \theta_i}{A(v)}$$

### 7.3 半边结构在共形算法中的应用

| 应用 | 所需操作 | 半边效率 |
|------|---------|---------|
| Ricci 流 | 边长变化、角度计算 | 快速邻接查询 |
| 调和映射 | 拉普拉斯-贝尔特拉米算子 | 1-环遍历 |
| 网格参数化 | UV 坐标传播 | 面遍历 |
| 曲率流 | 法向量更新 | 法向邻接 |

## 8. 关键公式

### 8.1 欧拉公式（网格）

$$V - E + F = \chi(M) = \begin{cases} 2 - 2g & \text{闭曲面，亏格 } g \\ 2 - 2g - b & \text{有 } b \text{ 个边界的亏格 } g \text{ 曲面} \end{cases}$$

### 8.2 离散高斯曲率（角度亏量）

$$K(v_i) = \frac{2\pi - \sum_{j} \theta_{ij}}{A_i}$$

其中 $\theta_{ij}$ 是顶点 $v_i$ 处第 $j$ 个三角形的内角，$A_i$ 是该顶点的局部面积。

### 8.3 Cotangent 权重 Laplacian

$$L_{ij} = \begin{cases} \sum_{k \neq i} \frac{1}{2}(\cot \alpha_{ik} + \cot \beta_{ik}) & i=j \\ -\frac{1}{2}(\cot \alpha_{ij} + \cot \beta_{ij}) & (i,j) \in E \\ 0 & \text{otherwise} \end{cases}$$

### 8.4 面积计算

**三角形面积** (Shoelace 公式):

$$A = \frac{1}{2} \sum_{i=1}^{n-1} (x_i y_{i+1} - x_{i+1} y_i)$$

**顶点 Voronoi 面积**（对偶面积）:

$$A_i = \frac{1}{4} \sum_{f \in \text{star}(i)} \sum_{e \in f} l_e^2 \cot \theta_e \quad \text{（优化形式）}$$

## 9. 核心洞察

### 洞察 1：半边 = 局部结构 + 全局连通

半边数据结构的核心思想是：**用局部的有向信息编码全局的拓扑结构**。每条半边只知道它的"局部关系"（起点、配对、同面下一条），但通过组合这些局部关系，我们可以推导出任意两点之间的拓扑连接。这与共形几何中"局部共形因子决定全局结构"的思想完全一致。

### 洞察 2：三角形的"稳定性"是离散化的基础

三角形是最简单的平面图形，其内角和恒为 $\pi$，边长决定角度（余弦定理）。这种确定性使得从连续几何到离散几何的转换（从光滑曲面到三角网格）具有良好的数学性质。特别是 Cotangent 权重 Laplacian 在尖锐特征附近可能不稳定，但整体上具有良好的收玫性。

### 洞察 3：1-环邻域是离散几何的"无穷小邻域"

在光滑流形上，我们在 $p$ 点的无穷小邻域内定义切空间、曲率等概念。在三角网格上，顶点 $v$ 的 **1-环邻域**（1-ring）恰好扮演了这个角色。1-环内的面和边决定了该顶点的局部几何——这正是离散微分几何的基本哲学。

### 洞察 4：数据结构的选择影响算法复杂度

| 操作 | 邻接表复杂度 | 半边复杂度 |
|------|------------|----------|
| 找顶点的所有相邻顶点 | $O(n)$ (遍历所有边) | $O(\deg(v))$ |
| 找边所属的两个面 | $O(n)$ | $O(1)$ |
| 遍历顶点的扇 | — | $O(\deg(v))$ |

选择半边结构，将许多看似复杂的问题转化为简单的线性遍历。

### 洞察 5：共形几何的离散化关键在于"角度-边长"关系

共形映射保持角度。在离散化时：
- **角度**直接存储在三角形中（由边长计算）
- **边长**由黎曼度量决定
- **共形因子** $e^{2u}$ 连接两者：$\cos \theta_{ij}' = e^{u_i+u_j} \cos \theta_{ij}$（共形变形后）

理解三角网格中角度与边长的关系，是设计共形几何算法的第一步。
